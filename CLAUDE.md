# CLAUDE.md — throughline-admin invariants

- **What this is**: internal ops console for the Throughline product family, live at www.throughline-admin.com (Vercel project `project-foundry/throughline-admin`, auto-deploys from `main`).
- **Design is law**: `design_handoff_throughline_admin/` holds the hi-fi spec and prototype. Colors, spacing, and microcopy are final (Brand Bible v2). Never introduce colors outside the token palette; never pure black/white. Creek Blue only for the brook strip and product-engagement bars.
- **Voice**: warm, unhurried path/water language. Never "unlock", "your journey", "seamlessly", "you're making great progress".
- **Database**: the `admin` schema of the shared Throughline Supabase project (`uuzzfeaevxilwizaittq`, us-west-1), role `throughline_admin`. The product's own tables live in `public` — this app must never touch them directly and the role has no grants there, with ONE approved exception: read-only views `admin.bridge_counts` and `admin.bridge_subscribers` (Myke-locked 2026-07-03; definition in `foundry/sql/bridge-views.sql`). Runtime connects through the transaction pooler (:6543, `pgbouncer=true`), migrations through the session pooler (:5432) — host `aws-1-us-west-1.pooler.supabase.com`.
- **Real data**: subscriber rows sync from the product via the bridge (product owns name/email + first-import values; steward owns plan/companion/status thereafter). Tiles read bridge counts with admin-schema fallback. Engagement/photos-per-subscriber/logs arrive only via telemetry. Chapters: the 7 real ones ("First Light" … "Last Night"); chapter totals are derived from the Chapter table, never hard-coded.
- **Statuses**: planned · active · invited · suspended · cancelled · dead (Myke-locked). Sync derives only invited/active; the rest are steward-set.
- **Schema changes**: Prisma migrations, checked in. `pnpm seed` = production bootstrap (idempotent, never resets steward password or heartbeating services). `pnpm seed:demo` = prototype demo dataset (destructive; demo counters 1,284 photos / 86 family / 412 events / 10 subscribers / 12 chapters).
- **Auth**: email + password against the `AdminUser` table (scrypt), JWT cookie, role `steward`. mykemiller@gmail.com is the sole steward; password is rotated in the Account screen. The seed creates the steward only if missing — it must never reset an existing password.
- **Telemetry**: all nine upstreams write through `POST /api/ingest/telemetry` with per-service bearer tokens (`SERVICE_TOKENS`). Contract documented in README — treat it as public API for product teams.
- **Env files**: never read or write `.env*` — template lives in `env.example`.
